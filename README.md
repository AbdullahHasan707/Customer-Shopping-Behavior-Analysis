# Customer Shopping Behavior Analysis

An end-to-end data analysis project exploring customer shopping behavior using **Python (Pandas)** for data cleaning, **SQL** for querying and analysis, and **Power BI** for interactive visualization. The project uncovers insights into spending patterns, discount usage, product ratings, shipping preferences, and subscription impact on revenue.

## Project Overview

Retail businesses generate huge volumes of transactional data, but the real value lies in turning that data into actionable insight. This project takes a raw customer shopping dataset (3,900 transactions) through a full analytics workflow:

1. **Clean & preprocess** the data in Python (Pandas)
2. **Load** it into a PostgreSQL database
3. **Query** it with SQL to answer key business questions
4. **Visualize** the findings in an interactive Power BI dashboard

## Dataset

The dataset (`customer_shopping_behavior.csv`) contains 3,900 customer transactions with 18 attributes, including:

- **Demographics:** Age, Gender, Location
- **Purchase details:** Item Purchased, Category, Purchase Amount (USD), Previous Purchases
- **Preferences:** Size, Color, Season
- **Engagement:** Review Rating, Subscription Status, Frequency of Purchases
- **Fulfillment:** Shipping Type, Payment Method, Discount Applied, Promo Code Used

## Data Cleaning & Preprocessing (Python)

Before analysis, the raw dataset was cleaned and enriched in [`Customer_Shopping_Behavior.ipynb`](./Customer_Shopping_Behavior.ipynb) using Pandas:

- **Missing values:** Filled missing `Review Rating` values with the median rating per product category
- **Column standardization:** Converted column names to lowercase, snake_case format for consistency
- **Feature engineering:**
  - Created an `age_group` column (Young Adult, Adult, Middle-aged, Senior) by binning customers into quartiles based on age
  - Created a `purchase_frequency_days` column, mapping categorical purchase frequency (e.g. Weekly, Monthly, Annually) to numeric day intervals for easier analysis
- **Redundancy removal:** Identified that `discount_applied` and `promo_code_used` were identical across all records and dropped the duplicate column
- **Database load:** Loaded the cleaned dataset into a PostgreSQL database using SQLAlchemy, ready for SQL analysis

## Business Questions & SQL Analysis

The analysis, documented in [`customer_behavior_project.sql`](./customer_behavior_project.sql), answers the following questions:

1. **Revenue by Gender** — What is the total revenue generated by male vs. female customers?
2. **Discount Behavior** — Which customers used a discount but still spent more than the average purchase amount?
3. **Top-Rated Products** — Which are the top 5 products with the highest average review rating?
4. **Shipping Comparison** — How does average purchase amount compare between Standard and Express shipping?
5. **Subscription Impact** — Do subscribed customers spend more? Comparing average spend and total revenue between subscribed and non-subscribed customers.

## Power BI Dashboard

The findings from the SQL analysis were brought to life in an interactive Power BI report (`customer_behavoir_Project.pbix`), featuring:

- Revenue breakdown by gender and category
- Customer segmentation by discount usage and spend
- Product performance by average review rating
- Shipping type comparison
- Subscription status vs. revenue and spend analysis

## Key Insights

- Spending patterns differ meaningfully between male and female customers.
- A segment of discount-using customers still spends above the average purchase amount, suggesting discounts aren't the only driver of high-value orders.
- Certain product categories consistently earn higher review ratings, highlighting opportunities for promotion.
- Shipping type correlates with differences in average order value.
- Subscription status has a measurable impact on both average spend and total revenue contribution.

## How to Use

1. Clone this repository.
2. Open `Customer_Shopping_Behavior.ipynb` in Jupyter to review or re-run the data cleaning and preprocessing steps.
3. Update the database credentials in the notebook (do **not** commit real credentials to GitHub — use environment variables or a `.env` file instead) and run it to load the cleaned data into PostgreSQL.
4. Open `customer_behavior_project.sql` in your preferred SQL client and run the queries against the loaded table.
5. Open `customer_behavoir_Project.pbix` in **Power BI Desktop** to explore the interactive dashboard.
